我的第一篇博客文章；使用Hexo框架创建博客

· 2020-7-8 · 次阅读

Person Blog（Win10平台）

1. 博客框架：Hexo

2. 安装框架：

2.1 安装nodejs（URL：nodejs.org）

2.1.1 下载nodejs

2.1.2 安装选好安装文件夹，一直下一步即可

2.2 检查是否安装成功

右键开始图标，选择power shell（管理员）

2.2.1 检查nodejs是否安装成功

依次输入以下命令

1. node -v
#检查node版本

2. npm -v 
#（手动注释# npm是包管理器）检查包管理器版本

3. npm install -g cnpm --registry=https://registry.npm.taobao.org
#从淘宝镜像源下载cnpm

4. cnpm -v
#检查cnpm版本，检验是否下载成功
*******************************
#注意，这一步可能出错，提示“禁止在此系统上运行脚本”
#解决办法是：
#    输入
    set-ExecutionPolicy RemoteSigned
#    命令，更改执行策略，然后输入A，确定更改即可
#再次输入cnpm -v，起飞飞飞飞飞飞飞飞飞飞飞飞……！

5. cnpm install -g hexo-cli
#主角来辽：安装hexo

6. hexo -v
#验证是否安装成功

***************
*OK！起飞！！！*
***************
实操过程如下：
↓↓↓↓↓↓↓↓↓↓

附安装过程截图

3. 新建博客项目并部署到github实现远程访问

3.1 准备工作

在当前目录下或者利用cd命令进入目标路径，新建blog（自定义）博客项目文件夹

1. cd h:
#进入h盘

2. md blog
#新建blog文件夹，用来存放博客项目资源

3. cd blog
#进入blog文件夹

4. pwd
#查看当前所处路径，是否是H:\blog

5. hexo init
#初始化hexo博客框架，他会从github上clone博客项目

6. ls
#查看blog目录下的资源文件，是否clone成功

7. hexo s
#或者hexo server
#或者hexo start
#启动进程，如下图

此时可以在浏览器打开http://localhost:4000，本地项目，端口号为4000

​ 当然，船新的博客会默认有一篇hello-world的博文。。。

1. #按提示操作，Press Ctrl+C to stop.
# 按Ctrl+C停止项目

2.hexo new "我的第一篇博客文章"
#新建一篇博客（会存储到H:\blog\source\_posts 目录下，利用cd命令进入即可查看；当然直接打开文件夹查看更香。。。）

3. ls
#H:\blog\source\_posts目录下即可查看当前目录下的文件夹以及文件

附.  cd ../..
#直接退回当前盘符的根目录
#cd后直接加完整路径也可直接到达该路径：cd H:\blog\source\_posts

4. 退回到blog目录，然后hexo clean
#清理一下

5. hexo generate
hexo g 是简写形式
#生成操作，会生成刚才新建的博文嵌入到项目中

6. hexo s
#再次启动
#打开浏览器，输入地址，🛫起飞飞飞飞飞……

6.1. Ctrl+C关闭
#没有什么stop是Ctrl+C解决不掉的，如果有，那就两次。。。或者Ctrl+C完了再Y一下。。。

7. E:\Typora\typora 我的第一篇博客文章.md
#当然，可以在ps（PowerShell）中使用其他编辑器命令打开博文进行编辑，这样会直接调用外部typora编辑器，编辑你的博文辣。。。
实际上，可以直接把打开文件夹编辑md博文，但是仍然建议用这样的方式来编辑，当然我只是建议。。。

3.2 安装git部署插件

3.2.1 在做这一步之前，需要到github新建一个仓库

但是此处的Repository name格式必须是：

昵称.github.io
#后续浏览器输这个地址即可访问博客

创建成功以后，会生成有一个空仓库，复制链接备用

ok
下一步
↓↓↓↓↓

3.2.2 用于部署到github，达到部署到远端的目的

  1.   1. cnpm install --save hexo-deployer-git
#安装git部署插件（注意：要在blog目录安装）

  2. 关键一步
     设置blog目录中的_config.yml文件，随便找个记事本啥的打开都行，找到尾部：把deploy设置一下：
     deploy:
       type: git
       repo: https://账户名:密码@github.com/Kerr-An/Kerr-An.github.io.git
       branch: master
       #repo这样设置的好处是使用“hexo d”命令部署的时候不需要输入账户和密码

3. hexo d
#安装之后就可以部署了d->deployer：部署

ok

4. 关于美化

4.1 更换主题

  • # yilia主题：地址栏键入
github.com/litten/hexo-theme-yilia

查看这个yilia主题
然后捏，再去powershell里，blog目录下，键入
git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
意思是使用git从github上克隆yilia主题到themes/yilia文件夹

clone下来之后，只需要修改blog目录下的_config.yml文件，找到theme，改成yilia，保存退出

然后hexo clean清理一下，然后hexo g生成一下，j接着hexo s启动，本地查看localhost:4000；然后Ctrl+C关闭；再然后hexo clean，hexo d。

至此，打完收工！！！
